Git Commands

Commun used Git Commnads and what they do.

:pushpin:Git init
  • Initialize a local Git repository
  • syntaxis: git init
:pencil2:Git Commit
  • Records a snapshot of the file permanently in the version history.

  • git commit -m 'comment explaining changes since last commit '

  • Git Push
    • Push changes to remote repository (remembered branch)

    • git pushvariableName branchName

  • Git Pull
    • Update local repository to the newest commit.

    • git pull https://github.com/example/example.git

  • Git Revert
    • Revert commmit changes.
  • Git Add
    • Saves your changes in a file to the staging area.

    • git add [file name]]

  • Git Clone
    • Copies a repository from an existing URL.

    • git clone repo_urlhttps://github.com/example/example.git

  • Git Merge
    • The history of the specified branch is merged into the current branch with this command.
    • Example:

      git merge branchName

  • Git Config
    • Sets the author name and email address to be used with the commit command.

    • git config --global user.name 'username '

    • git config --global user.email 'user@email.com '

  • Git Status
    • Check status (It will inform us of Gits thoughts and the state of our repository as seen by Git.) Lists all files, stating files that have changes to be committed, or files that need to be added to staging area.

    • git status

  • Git Remove
    • Removes individual files or a collection of files from the staging area and the working directory of the current branch.

    • git rm exampleFile.txt
